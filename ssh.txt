Работа с ssh
1. Установка доверенных отношений с сервером 
При первом соединении с сервером клиент получает публичный ключ сервера и пользователь джолжен подтвердить правильность публичного ключа
Сличение ключа можно провести с помощью сличения отпечатка (fingerprint) 
После подтверждения соединения на стороне клиента информация о публичном ключе сервера записывается в файл в папке пользователя ~/.ssh/known_hosts
При удалении этого файла доверительное соединение требуется установить заново.

2. Генерация ssh-ключей пользователем
ssh-keygen -t <тип алгоритма шифрования> -C <Электронная почта владельца ключа>
ssh-keygen -t ed25519 -C "ivanov@mail.ru"

Для двухфакторной защиты секретного ключа необходимо использовать парольную фразу (длиной более 12 символов)

3. Копирование публичного ключа пользователя на сервер
ssh-copy-id <имя_пользователя_на_сервере>@<адрес_сервера>
ssh-copy-id student@172.16.7.85
Ввести пароль от удаленной учетной записи

4. Добавление секретного ключа в клиента ssh
ssh-agent bash 
ssh-add <путь к файлу секретного ключа>
ssh-add ~/.ssh/id_ed25519

Ввести парольную фразу для однократного получения доступа к секретному ключу

5. После выполнения шагов со 2 по 5 соединение с сервером будет осуществляться без пароля 
(аутенфикация и идентификация проходит при помощи пары ключей секретный-публичный)
